2. 1주차 주요 개념 키워드 적어보기

- Git Github 차이
Git : 나의 컴퓨터에 사용하고, 버전관리하는 도구

Github : 원격으로 올릴 수 있는 클라우드(구글 드라이브 등등)
	    어떤 사이트에 들어가서 나의 프로젝트를 올릴 수 있는 기능을 제공
	    깃으로 커뮤니티 기능 같은 것들을 사용할 수 있다.

Github은 깃으로 관리되는 프로젝트가 저장되어 있는 공간을 제공하고
개발자 커뮤니티 기능을 하는 서비스.

- Sourcetree란 무엇인가?

Git 은 여러가지 방법으로 사용할 수 있다. 아래처럼 터미널에 명령어를 직접 입력할 수도 있다.(이런 방법을 CLI 라 부른다..

- Git 버전관리
깃은 커밋을 통해서 현재 프로젝트 상태를 저장하고 조회하는 것
누가, 언제, 현재 프로젝트 내용 어떤지 정보남기는 것!
누가(작성자), 언제(날짜), 현재프로젝트 내용(설명)

- commit이란? 
현재 프로젝트 상태 저장하는 것!
깃이 전 커밋 상태를 알고 있으니, 어디가 추가 됐고, 삭제 됐는지 확인 

- clone 
git clone 이란?
다른사람이 작업한 내용을 복제 하여 내 로컬영역에 복제품을 만드는 것

- push 
내가 작업하고 있는 내용을 깃헙이라는 협업 프로그램에 올릴수 있게 하는것!

- pull 
push와 다르게 pull은 같이 협업하고있는 사람의 업데이트 된 내용을 나의 로컬영역으로 당겨오는 것
